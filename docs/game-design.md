# Wave Defence 기획 문서

> 작성일: 2026-07-09
> 상태: 초안 — 기술 스택 확정, 게임 기획안 작성 예정

## 1. 프로젝트 개요

HTML5 기반 브라우저 게임 (2D 웨이브 디펜스). 설치 없이 URL로 바로 플레이할 수 있는 웹 배포를 목표로 한다.

## 2. 기술 스택

### 2.1 선정 결과

| 구성 요소 | 선택 | 버전 | 비고 |
|---|---|---|---|
| 게임 프레임워크 | Phaser | 4.2.x ("Giedi", 2026-06) | 물리(Arcade)·오디오·입력·씬 관리·파티클·트윈 내장 |
| 언어 | TypeScript | 5.9+ | Phaser에 타입 정의 내장 (@types 불필요) |
| 빌드 도구 | Vite | 8.x | HMR 지원 |
| 런타임 | Node.js | 22 LTS 이상 | Vite 요구사항: 20.19+ / 22.12+ |
| 테스트 (선택) | Vitest | 최신 | Vite와 통합 |
| 린트/포맷 (선택) | Biome | 최신 | ESLint+Prettier 대체, 소규모 프로젝트에 적합 |

런타임 의존성은 사실상 Phaser 하나. 웨이브 디펜스에 필요한 기능(물리, 사운드, 파티클 등)이 전부 내장되어 있어 추가 라이브러리가 거의 필요 없다.

### 2.2 선정 이유

- **Phaser**: HTML5 2D 게임의 사실상 표준. 커뮤니티·예제가 가장 풍부하고, 웹 전용 게임에서 로딩 속도와 배포 편의성이 가장 유리. Phaser 4는 WebGL 렌더러를 새로 작성한 최신 메이저 버전으로, Phaser 3와 API 호환성을 유지.
- 검토했던 대안:
  - **순수 Canvas/JS**: 의존성 제로지만 물리·씬 관리 등을 직접 구현해야 함
  - **PixiJS**: 더 가볍고 렌더링은 빠르지만 렌더러만 제공 (게임 로직 직접 구현)
  - **Godot 웹 익스포트**: 에디터 기반, 2D/3D 지원. 웹 빌드 용량이 큼
  - **Three.js / Babylon.js / PlayCanvas**: 3D용이라 제외
  - **Unity 웹 빌드**: 가벼운 브라우저 게임에는 과함

### 2.3 참고: 2026년 웹 게임 트렌드

- **WebGPU 전면 지원**: Safari 26(2025-09)을 마지막으로 모든 주요 브라우저에서 WebGPU 사용 가능. 브라우저 게임 성능 상한이 크게 상승
- **브라우저 배포 재부상**: 설치·런처·스토어 심사 없이 URL로 즉시 플레이 가능한 점 때문에 인디 개발자의 웹 배포 회귀 추세
- **WASM + Rust**: 고성능이 필요한 경우의 선택지 (본 프로젝트에는 불필요)

### 2.4 프로젝트 시작 방법

```bash
npm create @phaserjs/game@latest
```

프롬프트에서 Phaser 4 + Vite + TypeScript 선택 시 씬 구조가 잡힌 프로젝트가 생성됨.

### 2.5 향후 필요 시 추가 검토

- **UI 오버레이**: 점수판·메뉴는 Phaser 내장 텍스트/DOM으로 충분. 복잡해지면 공식 React 템플릿(template-react-ts) 검토
- **멀티플레이**: Colyseus (Phaser+Colyseus 공식 모노레포 템플릿 존재, 2026-06)
- **배포**: 정적 호스팅 (GitHub Pages / Vercel / itch.io)

## 3. 게임 기획안

### 3.1 컨셉

**코어 중심 그리드 방어 + 레벨업에 따른 그리드 확장 + 웨이브 간 로그라이트 드래프트.**

- 화면 중앙의 코어를 지킨다. 코어 주변의 **배치 그리드** 위에만 유닛·구조물을 배치할 수 있다
- 전투로 XP를 얻어 **레벨업하면 그리드가 한 겹씩 넓어져** 더 많은 배치물과 더 깊은 방어선을 만들 수 있다
- **유닛**은 웨이브 사이에 자유롭게 재배치할 수 있고, **구조물**은 페널티(부분 환급)를 감수하면 철거 후 재건설할 수 있다
- 한 판이 독립된 "런"이며, 웨이브 간 3택1 드래프트가 매 판 다른 빌드를 만든다 (2026년 TD+로그라이트 하이브리드 트렌드 부합)

### 3.2 레퍼런스

| 게임 | 참고할 점 |
|---|---|
| Kingdom Rush | 적은 종류의 배치물 + 깊은 업그레이드. 역할별 명확한 분담 |
| They Are Billions | 산발적 침입과 대규모 웨이브를 오가는 긴장 곡선 설계 |
| Dome Keeper | 웨이브 사이 페이즈에 의미 있는 선택을 넣는 구조 |

### 3.3 코어 루프

한 판(런) = 15~20분.

```
웨이브 전투 (골드·XP 획득, 레벨업 시 그리드 즉시 확장)
  → 클리어
  → 보상 드래프트 (3택1)
  → 배치 페이즈 (유닛 재배치 · 건설 · 철거 · 업그레이드, 시간 제한 없음)
  → "시작" 버튼 → 다음 웨이브   (반복 → 최종 웨이브/보스로 마무리)
```

- **승리**: 최종 웨이브(예: 20웨이브) 클리어
- **패배**: 코어 HP 0

### 3.4 핵심 시스템 (7가지)

1. **코어** — 화면 중앙, 단일 HP. 0이면 패배
2. **배치 그리드** — 코어를 중심으로 한 정사각 그리드(셀 64px). 그리드 안에서만 배치 가능. 시작 5×5, 레벨업마다 한 겹(링)씩 확장 (7×7 → 9×9 → 11×11). 그리드 확장 = 배치 슬롯 증가 + 방어선을 더 바깥에 칠 수 있는 전략 공간 확장
3. **레벨(XP)** — 적 처치로 XP 획득, 레벨업 시 그리드 확장. 웨이브 중에도 즉시 반영
4. **배치물 (유닛 2종 + 구조물 3종)**
   - **유닛** (검병·궁수): HP 보유. 웨이브 중 사망 시 해당 웨이브 동안 부재, 다음 배치 페이즈에 부활. 웨이브 사이 재배치는 무료·무제한
   - **구조물** (캐논·모르타르·프로스트): HP 보유. 파괴되면 소멸(재건설 필요). 철거 시 투자 비용의 50%만 환급 (= 재배치 페널티)
5. **적 + 타게팅** — 적은 코어를 향해 직진하되, 경로에서 유닛/구조물과 접촉하면 그것을 먼저 공격(공격형) 또는 그것에 자폭(자폭형). 배치가 곧 방어선이 된다
6. **경제** — 적 처치 시 골드. 용도는 배치/업그레이드. 통화는 골드 하나 (XP는 별도 축)
7. **웨이브 간 드래프트** — 웨이브 클리어 시 무작위 강화 카드 3택1. 가산/곱연산 스탯 수정이라 구현 비용이 낮고 재플레이성의 핵심

### 3.5 수치 초안 (밸런싱 테이블)

> 전부 플레이테스트로 조정될 출발점. 구현 시 코드가 아닌 데이터 테이블(TS const 또는 JSON)로 관리한다.

**전제**

- 해상도 1280×720, 코어는 중앙. 속도 단위: px/초
- 코어 HP **100**, 시작 골드 **100**, 시작 그리드 **5×5**

**레벨 테이블**

| 레벨 | 필요 누적 XP | 그리드 | 배치 가능 셀 수 |
|---|---|---|---|
| 1 (시작) | — | 5×5 | 24 (코어 1셀 제외) |
| 2 | 30 | 7×7 | 48 |
| 3 | 90 | 9×9 | 80 |
| 4 (최대) | 200 | 11×11 | 120 |

**적 스탯**

| 적 | 행동 | HP | 속도 | 피해 | 골드 | XP | 역할 |
|---|---|---|---|---|---|---|---|
| 일반 | 자폭형 | 20 | 60 | 5 (접촉 시 1회) | 5 | 1 | 기준치 |
| 고속 | 자폭형 | 10 | 120 | 3 (접촉 시 1회) | 4 | 1 | 슬로우/광역 없으면 새어나감 |
| 탱커 | 공격형 | 80 | 35 | 5 (1초 주기) | 15 | 3 | 단일딜 없으면 못 뚫음 |
| 보스 | 공격형 | 웨이브별 | 30 | 15 (1초 주기) | 100 | 15 | 5웨이브마다. DPS 체크 |

- **자폭형**: 첫 접촉 대상(유닛/구조물/코어)에 피해를 주고 즉시 소멸
- **공격형**: 접촉 대상을 일정 주기(`attackInterval`, 기본 1초)로 타격. 첫 타격도 접촉 후 한 주기가 지나야 발생(준비 시간). 대상을 파괴하면 다시 코어로 전진
- 웨이브 스케일링: 적 HP × `1 + 0.10 × (웨이브 − 1)`
- 보스 HP: W5 500 / W10 1,200 / W15 2,500 / W20 5,000

**배치물 스탯**

유닛 (웨이브 간 재배치 무료, 사망 시 다음 배치 페이즈 부활):

| 유닛 | 가격 | HP | 공격력 | 공속 | 사거리 | 역할 |
|---|---|---|---|---|---|---|
| 검병 | 30 | 60 | 8 | 1/초 | 근접 | 블로커. 전선 유지, 적 발 묶기 |
| 궁수 | 40 | 30 | 10 | 1/초 | 200 | 기동 딜러. 재배치로 화력 집중 |

구조물 (철거 시 50% 환급, 파괴 시 소멸):

| 구조물 | 가격 | HP | 공격력 | 공속 | DPS | 사거리 | 특수 |
|---|---|---|---|---|---|---|---|
| 캐논 | 70 | 100 | 25 | 0.8/초 | 20 | 250 | 단일 고딜 (대 탱커·보스) |
| 모르타르 | 90 | 80 | 10 | 0.5/초 | 5×N | 200 | 폭발 반경 60 (대 물량) |
| 프로스트 | 50 | 80 | 2 | 1/초 | 2 | 150 | 1.5초간 이속 −40% (유틸) |

- 업그레이드: 구조물만 3단계. 비용 = 기본가의 100% → 150% → 225%, 효과 = 단계당 공격력 +50% (프로스트는 감속률 +10%p). 유닛의 성장은 드래프트 카드로
- 검산: 시작 골드 100 = 검병 1 + 캐논 1. W1(일반 5 = 총 HP 100)을 캐논(20 DPS)이 검병이 발을 묶는 동안 처리하는 구도 → 첫 웨이브부터 "배치가 방어선"임을 학습

**웨이브 구성 (발췌)**

| 웨이브 | 구성 | 의도 |
|---|---|---|
| 1 | 일반 5 | 튜토리얼 |
| 2 | 일반 8 | 골드 모아 배치물 추가 |
| 3 | 일반 6 + 고속 4 | 고속 첫 등장 → 슬로우/블로커 학습 |
| 4 | 일반 8 + 고속 6 | |
| 5 | **보스** + 일반 5 | 첫 DPS 체크 |
| 6 | 일반 8 + 탱커 2 | 탱커 첫 등장 |
| 8 | 고속 12 | 물량 압박 → 모르타르 검증 |
| 10 | **보스** + 탱커 3 | |
| 11~19 | 혼합 비율 상승 (추후 상세화) | |
| 20 | **최종 보스** + 전 종류 혼합 | 클라이맥스 |

- 스폰 방향: W1~5 한 방향 → W6~ 2방향 → W15~ 전방향 (긴장 곡선). 방향이 바뀔 때 유닛 재배치의 가치가 극대화됨

**드래프트 카드 (12장, 등급 없음, 균등 확률, 중복 획득 가능)**

| 분류 | 카드 |
|---|---|
| 공격 (5) | 캐논 공격력 +25% · 모르타르 폭발 반경 +25% · 프로스트 감속률 +10%p · 궁수 공격력 +30% · 검병 공격력 +30% |
| 경제 (3) | 처치 골드 +20% · 즉시 골드 +100 · 건설 비용 −15% |
| 생존 (4) | 코어 HP 30 회복 · 유닛 최대 HP +30% · 구조물 최대 HP +30% · 적 전체 이속 −8% |

### 3.6 확정된 디자인 결정

| 항목 | 결정 |
|---|---|
| 배치 공간 | 코어 중심 그리드(64px 셀). 레벨업으로 확장 (5×5 → 11×11) |
| 배치물 이원화 | 유닛 = 웨이브 간 무료 재배치 / 구조물 = 철거 시 50%만 환급 |
| 적 행동 | 혼합형 — 일반·고속은 자폭형, 탱커·보스는 공격형. 경로상 배치물을 우선 대상화 |
| 웨이브 간 페이즈 | 시간 제한 없음. "시작" 버튼을 눌러야 다음 웨이브 개시 |
| 배치 방식 | 그리드 스냅 (코어 셀 및 점유 셀 제외) |

### 3.7 미확정 사항 (가정으로 채운 것 — 검증 필요)

- **레벨 곡선**: XP 요구치(30/90/200)와 최대 레벨 4는 임시값
- **유닛 사망 규칙**: "해당 웨이브 동안 부재 → 다음 배치 페이즈에 무료 부활"로 가정. 부활 비용을 받을지 여부 미정
- **구조물 철거 환급률**: 50%로 가정 (누적 업그레이드 비용 포함 기준)
- **적의 배치물 대상화 규칙**: "이동 경로와 접촉한 것만 공격"으로 가정. 일부러 우회하는 적 타입은 1차 범위 밖

### 3.8 1차 범위에서 제외

코어 루프의 재미가 검증된 뒤에 검토:

- 메타 진행 (런 간 영구 업그레이드)
- 멀티 맵 / 스테이지
- 플레이어 캐릭터 직접 조작
- 스토리
- 멀티플레이
- 모바일 터치 최적화

## 4. 화면 / UX

### 4.1 씬 구성 (Phaser Scene 매핑)

```
Boot (에셋 로드) → Title (시작) → Game (전투+배치) ⇄ Draft (3택1 오버레이)
                                      ↓
                                   Result (승리/패배 → 재시작)
```

- **Game 씬**: 내부 상태 `BUILD`(배치 페이즈) ⇄ `WAVE`(전투)를 오간다. 배치물이 유지되어야 하므로 씬 분리가 아닌 상태 전환으로 처리
- **HUD**: 별도 UI 씬을 Game 위에 병렬 실행 (게임 월드와 UI 분리, Phaser 표준 패턴)
- **Draft**: 오버레이 씬. 웨이브 클리어 직후 반투명 배경 + 카드 3장 → 선택 시 닫히고 BUILD로

### 4.2 HUD 배치

- **상단 바**: 코어 HP 바 · 웨이브 번호(n/20) · 골드 · 레벨/XP 게이지(다음 그리드 확장까지)
- **하단 배치 바**: 배치물 5종 버튼(가격 표시, 골드 부족 시 비활성). BUILD에서만 표시
- **웨이브 시작 버튼**: BUILD에서만 표시, 하단 우측에 크게

### 4.3 배치 페이즈(BUILD) 조작

| 조작 | 방식 |
|---|---|
| 건설 | 하단 바 클릭 → 고스트 프리뷰가 커서 추적 + 그리드 스냅 (가능 셀 초록 / 불가 빨강) → 클릭 확정, 우클릭·ESC 취소 |
| 유닛 재배치 | 드래그&드롭 (BUILD에서만). 드래그 중 스냅 프리뷰 |
| 구조물 관리 | 클릭 → 소형 팝업 (업그레이드 비용 / 철거 환급액 표시) |
| 사거리 표시 | 프리뷰 중·선택된 배치물에 반투명 원 |
| 그리드 표시 | BUILD에서만 라인 표시. 레벨업 확장 시 새 링 하이라이트 연출 |

**다음 웨이브 예고**: BUILD 중 화면 가장자리에 다음 웨이브의 스폰 방향 화살표 + 적 구성 아이콘 표시. **완전 정보 공개** — 이 게임의 재미는 기습 대응이 아니라 주어진 정보로 최적 배치를 찾는 퍼즐이며, 예고가 없으면 재배치가 도박이 된다.

### 4.4 전투(WAVE) 중

- 배치 바 숨김. **전투 중 건설·재배치 불가** (배치는 오직 BUILD에서만 — "준비 → 관전" 리듬을 명확히. 플레이테스트 후 재검토)
- 유닛 사망 시 해당 셀에 흐린 실루엣 표시 (다음 BUILD에 부활 예정 안내)
- 코어·배치물 피격 시에만 HP 바 잠깐 표시 (평소 숨김)

## 5. 구현 마일스톤

각 단계의 완료 판정 기준(DoD)을 만족해야 다음 단계로 넘어간다. 밸런스 튜닝은 M4 전까지 하지 않는다 (수치는 데이터 테이블 값 그대로).

| 단계 | 내용 | 완료 판정 기준 (DoD) |
|---|---|---|
| **M0 뼈대** | 그리드·코어 렌더, 캐논 배치, 일반 적 스폰·직진·자폭, 자동 사격, 패배 처리 | 브라우저에서 적을 막아내거나 코어가 터지는 것을 확인 가능 |
| **M1 코어 루프** | 웨이브 데이터 테이블, BUILD⇄WAVE 전환, 시작 버튼, 골드 경제, 배치물 5종, 적 4종(자폭/공격형), 정의된 웨이브 전부 클리어 시 승리 | 골드를 관리하며 웨이브 1~6을 클리어할 수 있고 승리/패배가 판정됨 |
| **M2 고유 시스템** | XP/레벨/그리드 확장, 유닛 드래그 재배치, 다음 웨이브 예고(방향+구성), 유닛 사망/부활 | 그리드 확장과 재배치가 실제 플레이 판단에 영향을 줌 |
| **M3 런 완성** | 드래프트 3택1, 구조물 업그레이드/철거(50% 환급), 20웨이브 전체 테이블, 보스 웨이브별 HP | 한 판을 15~20분에 완주 가능 |
| **M4 폴리시** | 연출(피격·사망·확장 하이라이트), 사운드, 밸런스 튜닝 | 외부인에게 보여줄 수 있는 상태 |

> **1차 프로토타입 목표 = M0~M2** (드래프트·업그레이드 없이 코어 루프 + 고유 시스템까지)

wave-defence/

├── index.html

└── src/
    ├── main.ts            # Phaser Game 설정, 씬 등록
    ├── data/

    ├── scenes/

    │   ├── TitleScene.ts
    │   ├── GameScene.ts   # BUILD ⇄ WAVE 상태 머신, 배치·전투
    │   ├── UIScene.ts     # HUD (Game 위에 병렬 실행)
    │   └── (DraftScene.ts, M3에서 추가)
    ├── entities/

## 8. 아트 · 사운드 방향

- **프로토타입(M0~M3)**: 색 도형만 사용 (Phaser Shape). 팔레트 — 적: 붉은 계열, 아군 유닛: 파랑/초록, 구조물: 남색/주황/하늘색, 코어: 노랑
- **검증 후(M4)**: 무료 에셋으로 교체 검토 (Kenney.nl 등 CC0 에셋). 픽셀 아트 vs 벡터는 이때 결정
- **사운드**: M4에서 추가 (Kenney 오디오 팩 또는 jsfxr류 생성 효과음). BGM은 선택 사항
